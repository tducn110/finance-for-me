	6. COURSE CONTENT
This course primarily uses the textbook "Fundamentals of Web Development," third edition, by Randy Connolly & Ricardo Hoar for references. In addition, we adopt more modern and popular tools and best practices to provide students with practical skills and insights that are directly applicable in professional settings.
SECTION 1. INTRODUCTION
In this section, we'll delve into the foundational concepts of web development. We'll explore what the web is, its significance, and the various types of web architectures. Additionally, we'll discuss different approaches to web development, including low code, no code, and custom development, preparing students for the journey ahead in building websites.
* Definition and History
* The Client-Server Model
* Where is the Internet
* Working in Web Development
* How to build an website (low code, no code, custom development)
SECTION 2. HOW THE WEB WORKS
This section provides a deep dive into the inner workings of the internet and the technologies that power it. From understanding internet protocols like TCP/IP, DNS, URLs, and HTTP, to exploring the roles of web browsers, servers, and databases, students will gain insight into the mechanics of how information is transmitted and accessed on the web.
* Internet Protocol (TCP/IP)
* Domain Name System (DNS)
* Uniform Resource Locator (URL)
* Hypertext Transfer Protocol (HTTP)
* Web Browsers
* Web Servers
   * Web Server
   * Web Application
   * Database Server
* Static vs Dynamic Web
SECTION 3. BUILD A STATIC WEB
Dive into the practical side of web development by learning to build static websites. Students will gain hands-on experience with HTML for content structure, CSS for styling, and JavaScript for adding interactivity. By the end of this section, students will have the skills to create visually appealing and functional static web pages.
* HTML & CSS
   * HTML & CSS Fundamentals
   * Selectors and Basic Styling
   * Table and Forms
   * Web Media
   * Layout & Responsive Design
   * CSS Effects (Transforms, Filters, Transitions, Animations)
* Javascript
   * Javascript Language Fundamentals
   * The Document Object Model (DOM)
   * Working with Events
   * Browser APIs
SECTION 4. TOOLS & BEST PRACTICES
In this crucial section, we'll explore the essential tools and best practices that streamline the web development process. From design tools like Figma to version control systems like Git, we'll cover a range of tools for design, development, testing, deployment, and maintenance. By adopting industry-standard tools and best practices, you'll optimize your workflow and ensure project success.
Phase
	Tools
	Design
	* Design System: Offering options like Material, Ant, and Fluent provides flexibility for students to explore different design aesthetics and component libraries, which can enrich their understanding of design principles and user interface (UI) design.
* Design Tool (Figma): Its popularity, collaborative features, and robust design capabilities. It's widely used in the industry, ensuring that students gain practical skills applicable in real-world scenarios.
	Development 
	* Build tools (Vite): Vite's emphasis on speed and simplicity aligns well with modern web development practices, enabling students to experience rapid feedback cycles and efficient development workflows.
* CSS Frameworks (Tailwind CSS): Tailwind CSS's utility-first approach offers a unique perspective on CSS development and encourages modular, scalable UI design. It's a valuable addition to students' skill sets, complementing traditional CSS frameworks.
* Javascript Frameworks (Vue.js): Vue.js is a popular and versatile JavaScript framework known for its ease of use and progressive nature. Its growing adoption in the industry makes it a valuable skill for students to acquire, particularly for building dynamic and interactive web applications.
* Version Control (Git, GitHub): Git and GitHub are industry-standard tools for version control and collaboration. Familiarizing students with these tools equips them with essential skills for working in development teams and contributing to open-source projects.
	Test
	* Automation Test (Cypress): Cypress is an excellent choice for end-to-end testing, offering a user-friendly testing experience and powerful features. It enables students to write reliable tests and gain confidence in the quality of their code.
* SEO Metrics (Google Lighthouse): Google Lighthouse provides valuable insights into website performance and SEO optimization, helping students understand the importance of optimizing web content for search engine visibility and user experience.
	Deploy
	* CI/CD (GitHub Actions): GitHub Actions streamlines the deployment process with automated workflows, enhancing efficiency and reliability. It's a practical tool for students to learn about continuous integration and deployment practices.
	Maintains
	* Tracking & Analytics (Google Analytics & Tag Managers): GA & GTM Offer comprehensive tracking and analytics capabilities, providing students with valuable insights into website usage and user behavior. These tools are essential for monitoring and optimizing website performance.
* Application Logs & Monitoring (Sentry): Sentry is an indispensable tool for error tracking and monitoring, helping identify and resolve issues in real-time. It promotes proactive maintenance and ensures a seamless user experience.
	SECTION 5. BUILD A DYNAMIC WEB
Building on the foundational knowledge gained in previous sections, we'll now shift our focus to dynamic web development. You'll learn server-side programming with Node.js and explore the differences between single-page applications (SPA), multi-page applications (MPA), and static site generators (SSG). By mastering dynamic web development techniques, students will be equipped to create interactive and data-driven web applications that meet modern user demands.
* What is Server-Side Development
* Databases
* Server Side Programing
* SPA vs MPA vs SSG
* REST API & GraphQL API
EXTENDED TOPICS
